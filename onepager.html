ship

autonomous coding agent

design in, working code out. no supervision required.

ship reads your design specifications, breaks them into executable tasks, and autonomously crafts working software through parallel execution. No supervision required. No babysitting. No "one more thing."

planner-worker-judge pattern on the command line. proven at scale by cursor, packaged for your terminal.

what it does

# create your design
cat > design.txt <<EOF
- create fastapi server with /health endpoint
- add GET /users endpoint with pagination
- add POST /users endpoint with validation
- write pytest tests for all endpoints
- add openapi documentation
EOF

# let ship build it
uvx github.com/kronael/ship design.txt

# result: complete API server, tests, docs
# built autonomously while you do other work

core principles

parallel execution

4 workers execute tasks simultaneously. because waiting is for mortals.

project isolation

each project has its own .ship/ state. run multiple instances. no conflicts.

continuation

interrupted mid-build? ship -c picks up exactly where it left off.

autonomous

runs to completion and exits. creates the project, leaves detailed notes, doesn't hover.

architecture

implements cursor's planner-worker-judge pattern, proven at production scale

planner

reads your design, breaks it into atomic tasks. thinks once, clearly, at startup. no overthinking.

workers

4 parallel executors via Claude Code CLI. full tool access with --permission-mode acceptEdits.

judge

checks completion every 5s. when satisfied, exits cleanly. the most decisive code review you'll ever get.
